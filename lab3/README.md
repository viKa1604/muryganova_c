Цель работы: изучить работу с динамической памятью: функции malloc, calloc, realloc,
free, создание и использование динамических массивов.

Теория:
malloc — выделяет память
calloc — выделяет и заполняет нулями
realloc — изменяет размер
free — освобождает память
Программист обязан управлять памятью вручную и избегать ошибок, связанных с

5.Зачем нужен free

Ответы на вопросы:

1.Что делает malloc?
Выделяет блок памяти заданного размера в куче. Возвращает указатель 
на выделенную память. При ошибке возвращает NULL.

2.Чем отличается malloc от calloc?
- malloc — выделяет память, но не обнуляет её (содержит мусор)
- calloc — выделяет память и обнуляет все байты

3.Что делает realloc?
Изменяет размер ранее выделенного блока памяти. Может увеличить 
или уменьшить его, при необходимости перемещая данные в новое место.

4.Что произойдет при утечке памяти?
Программа будет потреблять память без освобождения, что может привести 
к замедлению работы, исчерпанию памяти и аварийному завершению.

5.Зачем нужен free?
Освобождает динамически выделенную память, возвращая её системе. 
Предотвращает утечки памяти.

6.Что произойдет при двойном освобождении памяти?
Неопределенное поведение. Программа может аварийно завершиться 
или повредить менеджер памяти.

7.Как проверить, что malloc вернул NULL?
int *arr = (int*)malloc(100 * sizeof(int));
if (arr == NULL) {
    printf("Ошибка выделения памяти!\n");
}

8.В чем отличие статического и динамического массива?

Статический: Размер фиксирован на этапе компиляции;
Выделяется в стеке;
Освобождается автоматически;
int arr[10];
Динамический: Размер можно изменить во время выполнения;
Выделяется в куче;
Требуется ручное освобождение (free);
int *arr = malloc(10 * sizeof(int)).
